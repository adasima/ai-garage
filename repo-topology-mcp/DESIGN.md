# Repo Topology & Metrics MCP (`repo-topology-mcp`)

## 目的
プロジェクト全体のファイル間インポート・依存関係を解析し、依存関係グラフの構築と複雑度の計測を行う。
大規模なリポジトリにおいても、エージェントが「どのファイルを修正すべきか」「影響範囲はどこか」を自律的に判断できるようにする。

## 提供する機能 (Tools)
- `get_dependency_graph`: 特定のファイルを起点に、そのファイルが依存している「依存先ファイル」と、呼び出している「依存元ファイル」の一覧を返す。
- `get_hotspots`: 複雑度が高い、または変更頻度が高いファイルをリストアップする。

## 実装方針
- 特定のディレクトリ（`src` や `src-tauri` 等）をトラバースし、import / use 構文を抽出する。
- キャッシュ機能を持たせ、大規模なプロジェクトでも高速に応答できるようにする。
